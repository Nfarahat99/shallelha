# Plan 08-02: Rate Limiting & Input Hardening

## Goal
Room creation and join are rate-limited per socket/IP; O(n) Redis `keys('room:*')` scan eliminated; host impersonation mitigated; socket event counters cleaned up on disconnect.

## Prerequisites
None — this plan can start in parallel with Plans 08-01, 08-03, 08-04. No shared files.

## Threat Model
<threat_model>
| Threat | STRIDE | Mitigation |
|--------|--------|------------|
| room:create spam exhausts room codes | Denial of Service | Per-socket rate limiter: 5 room:create per 60s |
| room:join brute-force guesses room codes | Denial of Service | Per-IP rate limiter: 10 room:join per 60s |
| Malformed room code crashes Redis lookup | Tampering | Regex validation `/^[A-HJ-NP-Z]{4}$/` before any Redis call |
| O(n) `redis.keys('room:*')` scan blocks event loop | Denial of Service | Redis reverse index `hostroom:{userId}` set on createRoom, O(1) lookup |
| Player impersonates host by sending fake userId | Elevation of Privilege | Cross-check `socket.handshake.auth.userId` against room's stored `hostId` |
| socketEventCounts Map grows unbounded from disconnected sockets | Denial of Service | `disconnect` handler calls `socketEventCounts.delete(socket.id)` |
</threat_model>

## Tasks

### Task 1: Create per-socket event rate limiter utility
**File(s):** `apps/server/src/socket/middleware/rateLimiter.ts`
**What:**
- Create `apps/server/src/socket/middleware/rateLimiter.ts`.
- Export `socketEventCounts: Map<string, Map<string, { count: number; resetAt: number }>>()` — outer key is `socket.id`, inner key is event name.
- Export function `checkRateLimit(socketId: string, event: string, limit: number, windowMs: number): boolean`. Returns `true` if allowed, `false` if rate-limited. On first call for a socket+event, creates entry with count=1, resetAt=`Date.now()+windowMs`. Subsequent calls increment count; if count > limit AND now < resetAt, return false. If now >= resetAt, reset count to 1 and resetAt to now+windowMs.
- Export function `clearSocketRateLimits(socketId: string): void` — deletes outer map entry for the socket.
- No external dependencies — pure in-memory (single-instance server, no Redis needed).
**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. File exports `checkRateLimit`, `clearSocketRateLimits`, `socketEventCounts`.

### Task 2: Apply rate limiter to room:create (5/min per socket)
**File(s):** `apps/server/src/socket/room.ts`
**What:**
- Import `checkRateLimit` from `./middleware/rateLimiter`.
- At the top of the `room:create` handler (before `getAuthUserId`), call `checkRateLimit(socket.id, 'room:create', 5, 60_000)`. If returns `false`, emit `room:error` with message `'Rate limit exceeded — try again later'` and `return`.
**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. `grep -c "checkRateLimit" apps/server/src/socket/room.ts` returns >= 1.

**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. `grep "room:join" apps/server/src/socket/room.ts | grep -c "checkRateLimit"` returns >= 1 (or verify via grep that checkRateLimit appears in the room:join handler block).

### Task 4: Add room code format validation
**File(s):** `apps/server/src/socket/room.ts`
**What:**
- In the `room:join` handler, after the existing `if (!roomCode || !name || !emoji)` check, add: `if (!/^[A-HJ-NP-Z]{4}$/.test(roomCode)) { socket.emit('room:error', { message: 'Invalid room code format' }); return }`. This matches the `generateCode` charset (excludes I and O), preventing invalid codes from hitting Redis.
- Apply the same regex check in the `reconnect:player` handler before the `getRoom` call.
**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. `grep -c "A-HJ-NP-Z" apps/server/src/socket/room.ts` returns >= 1.

### Task 5: Add Redis reverse index for host room lookup
**File(s):** `apps/server/src/room/room.service.ts`
**What:**
- In `createRoom()`, after `await redis.expire(roomKey(code), ROOM_TTL)`, add: `await redis.set(\`hostroom:${hostId}\`, code, 'EX', ROOM_TTL)`. This creates a reverse index from hostId to room code.
- Rewrite `findRoomByHostId(hostId)` to: `const code = await redis.get(\`hostroom:${hostId}\`); if (!code) return null; return getRoom(code)`. This is O(1) instead of O(n) `redis.keys('room:*')` scan.
- In `updateRoomStatus()`, when status is `'ended'`, add: `const raw = await redis.hgetall(roomKey(code)); if (raw?.hostId) await redis.del(\`hostroom:${raw.hostId}\`)`. This cleans up the reverse index when a game ends.
**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. `grep -c "hostroom:" apps/server/src/room/room.service.ts` returns >= 2. `grep -c "redis.keys" apps/server/src/room/room.service.ts` returns 0 (O(n) scan removed).

### Task 6: Add host identity cross-verification
**File(s):** `apps/server/src/socket/room.ts`
**What:**
- In the `room:start` handler, after the `if (!socket.data.isHost)` check and after obtaining `roomCode`, add: `const room = await getRoom(roomCode); if (!room || room.hostId !== socket.data.userId) { socket.emit('room:error', { message: 'Host verification failed' }); return }`. This cross-checks the socket's userId against the stored room hostId.
- Apply the same cross-check pattern in `room:end` handler: fetch room, verify `room.hostId === socket.data.userId`.
**Acceptance:** `cd apps/server && npx tsc --noEmit` passes. `grep -c "room.hostId !== socket.data.userId" apps/server/src/socket/room.ts` returns >= 2.
