# Plan 08-04: Production Configuration & Deployment Audit

## Goal
All environment variables documented and verified; E2E test scripts use env vars instead of hardcoded URLs; `next.config.mjs` remotePatterns confirmed; `robots.txt` added; Railway and Vercel deployment configs audited.

## Prerequisites
None — this plan can start in parallel with Plans 08-01, 08-02, 08-03. No shared files.

## Threat Model
<threat_model>
| Threat | STRIDE | Mitigation |
|--------|--------|------------|
| Secrets committed to repo via `.env` | Information Disclosure | `.env.example` files with placeholder values only; `.gitignore` already excludes `.env` |
| E2E tests hit wrong environment | Tampering | Replace hardcoded URLs with `process.env.BASE_URL` / `process.env.SERVER_URL` with fallback defaults |
| Search engines index unfinished pages | Information Disclosure | `robots.txt` allows all (MVP is public); revisit if needed |
| NEXT_PUBLIC_ vars expose server secrets | Information Disclosure | Audit: only `NEXT_PUBLIC_SOCKET_URL` should be public; no API keys or secrets in NEXT_PUBLIC_ |
</threat_model>

## Wave 0 -- Test Stubs
No new test files. Verification is via build success + file existence checks. Existing tests must still pass:
- `cd apps/server && npm run test`

## Tasks

### Task 1: Audit and create web .env.example
**File(s):** `apps/web/.env.example`
**What:**
- Read all files in `apps/web/` that reference `process.env` or `NEXT_PUBLIC_`. Use `grep -r "process.env\|NEXT_PUBLIC_" apps/web/` to find all references.
- Create `apps/web/.env.example` listing every env var with a descriptive placeholder comment. Expected vars:
  ```
  # Backend Socket.io URL (public — exposed to browser)
  NEXT_PUBLIC_SOCKET_URL=http://localhost:4000
  ```
- Verify NO secret values (API keys, database URLs) appear in `NEXT_PUBLIC_` vars. If any are found, flag them in a comment: `# WARNING: This should NOT be NEXT_PUBLIC_ — move to server-side`.
**Acceptance:** `apps/web/.env.example` exists. `grep -c "NEXT_PUBLIC_" apps/web/.env.example` returns >= 1.

### Task 2: Audit and create server .env.example
**File(s):** `apps/server/.env.example`
**What:**
- Read all files in `apps/server/src/` that reference `process.env`. Use `grep -r "process.env" apps/server/src/` to find all references.
- Create `apps/server/.env.example` listing every env var with descriptive placeholder. Expected vars:
  ```
  # Server port
  PORT=4000

  # PostgreSQL connection string
  DATABASE_URL=postgresql://user:password@localhost:5432/shallelha

  # Redis connection string
  REDIS_URL=redis://localhost:6379

  # Frontend URL for CORS
  FRONTEND_URL=http://localhost:3000

  # Admin dashboard password
  ADMIN_PASSWORD=change-me-in-production

  # Node environment
  NODE_ENV=development
  ```
- Verify no actual secrets are in the file — only placeholder values.
**Acceptance:** `apps/server/.env.example` exists. `grep -c "DATABASE_URL\|REDIS_URL\|FRONTEND_URL\|PORT" apps/server/.env.example` returns >= 4.

### Task 3: Update e2e-test.mjs to use env vars
**File(s):** `e2e-test.mjs`
**What:**
- Replace the hardcoded lines at the top:
  ```
  const BASE = 'https://shallelha.vercel.app'
  const SERVER = 'https://shallelha-server-production.up.railway.app'
  ```
  With:
  ```
  const BASE = process.env.BASE_URL || 'https://shallelha.vercel.app'
  const SERVER = process.env.SERVER_URL || 'https://shallelha-server-production.up.railway.app'
  ```
- The defaults ensure the script still works without env vars (backwards compatible).
**Acceptance:** `grep -c "process.env.BASE_URL" e2e-test.mjs` returns 1. `grep -c "process.env.SERVER_URL" e2e-test.mjs` returns 1.

### Task 4: Update e2e-phase5.mjs to use env vars
**File(s):** `e2e-phase5.mjs`
**What:**
- Apply the same pattern as Task 3:
  ```
  const BASE = process.env.BASE_URL || 'https://shallelha.vercel.app'
  const SERVER = process.env.SERVER_URL || 'https://shallelha-server-production.up.railway.app'
  ```
**Acceptance:** `grep -c "process.env.BASE_URL" e2e-phase5.mjs` returns 1. `grep -c "process.env.SERVER_URL" e2e-phase5.mjs` returns 1.

### Task 5: Verify next.config.mjs remotePatterns
**File(s):** `apps/web/next.config.mjs`
**What:**
- Read the current `next.config.mjs` and verify `images.remotePatterns` includes Cloudinary (`res.cloudinary.com`).
- This is already present. No change needed if correct. If missing, add it.
- Add `output: 'standalone'` if not already present — this optimizes the Vercel deployment by producing a self-contained build.
- This task is a verification + minor optimization. If `output: 'standalone'` causes issues with Vercel (Vercel auto-detects Next.js and does not need standalone), skip it and leave a comment noting it was considered.
**Acceptance:** `grep -c "cloudinary" apps/web/next.config.mjs` returns >= 1. `cd apps/web && npx next build` succeeds.

### Task 6: Add robots.txt
**File(s):** `apps/web/public/robots.txt`
**What:**
- Create `apps/web/public/robots.txt` with:
  ```
  User-agent: *
  Allow: /

  Sitemap: https://shallelha.vercel.app/sitemap.xml
  ```
- The sitemap URL is a placeholder — no sitemap.xml is generated yet, but the reference is standard practice.
- Next.js App Router serves files from `public/` at the root path automatically.
**Acceptance:** `apps/web/public/robots.txt` exists. `grep -c "User-agent" apps/web/public/robots.txt` returns 1.

### Task 7: Verify Railway deployment config
**File(s):** `apps/server/Dockerfile` (read-only verification), `apps/server/railway.json` (create if missing)
**What:**
- Read `apps/server/Dockerfile` and verify it builds and runs the server correctly.
- If `apps/server/railway.json` does not exist, create it with:
  ```json
  {
    "$schema": "https://railway.app/railway.schema.json",
    "build": { "builder": "DOCKERFILE", "dockerfilePath": "Dockerfile" },
    "deploy": {
      "healthcheckPath": "/health",
      "healthcheckTimeout": 30,
      "restartPolicyType": "ON_FAILURE",
      "restartPolicyMaxRetries": 3
    }
  }
  ```
- If it already exists, verify `healthcheckPath`, `restartPolicyType`, and `restartPolicyMaxRetries` are present. Add any missing fields.
**Acceptance:** `apps/server/railway.json` exists with `healthcheckPath: "/health"`. `grep -c "healthcheckPath" apps/server/railway.json` returns 1.

### Task 8: Verify Vercel deployment config
**File(s):** `apps/web/vercel.json` (create if missing)
**What:**
- If `apps/web/vercel.json` does not exist, create it with:
  ```json
  {
    "framework": "nextjs",
    "buildCommand": "npx next build",
    "outputDirectory": ".next"
  }
  ```
- If it already exists, verify `framework` is `"nextjs"`. Vercel auto-detects most settings, so this is minimal.
- This is a safety net — Vercel works without `vercel.json` for Next.js projects, but having it documents the expected config.
**Acceptance:** `apps/web/vercel.json` exists. `cd apps/web && npx next build` succeeds.
