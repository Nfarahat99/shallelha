AFTER the emit call (after the closing `})` of the emit), add the following fire-and-forget analytics block. The prisma import should already exist in this file (check top of file for existing prisma import; if not present, add `import { prisma } from '../lib/prisma'` or match the existing import pattern used elsewhere in the server codebase — check `apps/server/src/lib/prisma.ts` or wherever prisma is imported from in this file).

```typescript
  // --- Analytics: fire-and-forget (ADMIN-04) ---
  // MUST NOT block the reveal — void + .catch() ensures no impact on game loop
  if (questionData.id) {
    // Increment timesPlayed
    void prisma.question.update({
      where: { id: questionData.id },
      data: { timesPlayed: { increment: 1 } },
    }).catch((err) => {
      console.error('[Analytics] timesPlayed increment failed:', err)
    })

    // Count wrong answers from playerStates and increment timesAnsweredWrong
    const wrongCount = Object.values(gameState.playerStates)
      .filter((s) => s.answeredCurrentQ && s.streak === 0)
      .length
    if (wrongCount > 0) {
      void prisma.question.update({
        where: { id: questionData.id },
        data: { timesAnsweredWrong: { increment: wrongCount } },
      }).catch((err) => {
        console.error('[Analytics] timesAnsweredWrong increment failed:', err)
      })
    }
  }
```

IMPORTANT NOTES:
- Use `void` prefix + `.catch()` — this is fire-and-forget. NEVER use `await` here.
- The `s.streak === 0` check detects players who answered wrong: when a player answers wrong, their streak is set to 0. Players who answered correctly have streak >= 1. Players who did NOT answer have `answeredCurrentQ === false` and are filtered out.
- Use Prisma atomic `{ increment: N }` — NEVER read-then-write.
- Place this AFTER the `io.to(roomCode).emit(...)` call, still inside the handleReveal function body.
- If `questionData.id` does not exist in the questionCache type, check what fields are available. The question data comes from `questionCache.get(roomCode)?.[gameState.currentQuestionIndex]`. If the cache stores the DB question object, it will have `id`. If it does not have `id`, add the `id` field to wherever the cache is populated (look for where questions are fetched from DB and stored in questionCache — likely in the game:start handler). The question query is `prisma.question.findMany({ where: { categoryId, status: QuestionStatus.approved } })` which returns objects with `id`.

**Step 2 — Create admin analytics Express route (apps/server/src/routes/admin.ts):**

Create `apps/server/src/routes/admin.ts` following the pattern from `health.ts`:

```typescript
import { Router } from 'express'
import rateLimit from 'express-rate-limit'
import { prisma } from '../lib/prisma'

const adminLimiter = rateLimit({
  windowMs: 60 * 1000, // 1 minute
  max: 30, // 30 requests per minute
  standardHeaders: true,
  legacyHeaders: false,
  message: { error: 'Too many requests, try again later' },
})

export const adminRouter = Router()

// GET /admin/analytics — question stats with wrong answer rates
adminRouter.get('/analytics', adminLimiter, async (_req, res) => {
  try {
    const questions = await prisma.question.findMany({
      select: {
        id: true,
        text: true,
        status: true,
        type: true,
        timesPlayed: true,
        timesAnsweredWrong: true,
        category: {
          select: { id: true, name: true },
        },
      },
      orderBy: { timesPlayed: 'desc' },
    })

    const stats = questions.map((q) => ({
      ...q,
      wrongRate: q.timesPlayed > 0
        ? Math.round((q.timesAnsweredWrong / q.timesPlayed) * 100) / 100
        : 0,
    }))

    res.json({ questions: stats })
  } catch (err) {
    console.error('[Admin] analytics error:', err)
    res.status(500).json({ error: 'Failed to fetch analytics' })
  }
})
```
