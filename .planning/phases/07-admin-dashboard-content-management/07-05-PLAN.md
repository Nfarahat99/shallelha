**Step 1 — Add composite unique constraint to Question model in schema.prisma:**

Read `apps/server/prisma/schema.prisma`. In the `Question` model, add a composite unique constraint at the bottom of the model (before the closing `}`):

```prisma
  @@unique([text, categoryId])
```

This enables `prisma.question.upsert({ where: { text_categoryId: { text, categoryId } } })` for idempotent seeding. The constraint ensures no duplicate questions within the same category.

      where: { slug: cat.slug },
      update: { name: cat.name },

      if (result.createdAt.getTime() === result.updatedAt.getTime()) {
        totalCreated++
      } else {
        totalSkipped++
      }
