<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → /packs/create Server Action | Pack data from unauthenticated or session-spoofed request |
| Browser → /api/ai/pack-generate | Proxy to Express AI route — could be called directly |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-02-01 | Spoofing | createPack Server Action — createdBy could be spoofed via direct POST | mitigate | Server Action reads session.user.id from auth() — never trusts client-supplied createdBy |
| T-10-02-02 | Tampering | submitPackForReview — any pack ID can be submitted | mitigate | Server Action checks pack.createdBy === session.user.id before status change |
| T-10-02-03 | Elevation of Privilege | /api/ai/pack-generate proxy bypasses rate limiter if called directly | accept | Rate limiter is on the Express route — proxy calls still go through it; no bypass |
</threat_model>
