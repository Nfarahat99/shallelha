## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03-01 | Denial of Service | Groq free tier quota exhaustion | mitigate | Rate limiter 3 req/host/hour; Groq free tier is 14,400 req/day — even 1,000 concurrent hosts at 3/hr = 3,000 req/hr, well within limits |
| T-10-03-02 | Tampering | Prompt injection via user-supplied prompt string | mitigate | System prompt is fixed and prepended; user prompt is the last message only; max 200 char limit reduces attack surface |
| T-10-03-03 | Information Disclosure | GROQ_API_KEY in server environment | accept | Key lives only in apps/server/.env (never web env); Railway secrets management; same pattern as OPENAI_API_KEY |
| T-10-03-04 | Spoofing | Rate limiter keyed on NextAuth session cookie — cookie can be spoofed | accept | Groq is free tier; worst case is quota drain by a determined attacker; acceptable risk for MVP; upgrade to DB-backed session check in Phase 12 (User Profiles) |
</threat_model>
