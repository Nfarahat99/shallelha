## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, Arabic letter labels inside option buttons |
| sm | 8px | Gap between elements within a card or button group |
| md | 16px | Default element spacing, grid gaps |
| lg | 24px | Section padding, answer feed row padding |
| xl | 32px | Layout gaps between major zones |
| 2xl | 48px | Major section breaks (host screen zones) |
| 3xl | 64px | Page-level spacing (not common in game screens) |

Exceptions:
- Touch targets on player phone: min-h-[80px] and min-w-[44px] on all tappable buttons (iOS Safari 44px minimum — existing pattern from AnswerOptions.tsx)
- Free text textarea: min 3 rows height (natural content height, not a spacing token)
- Host top bar: h-12 (48px) — matches existing HostGameScreen.tsx pattern
- Host controls bar (bottom): py-3 px-6 — matches existing HostInGameControls.tsx pattern
- iOS safe area bottom: `pb-[env(safe-area-inset-bottom)]` on PlayerGameScreen wrapper — existing pattern

Source: codebase read — `AnswerOptions.tsx`, `HostGameScreen.tsx`, `HostInGameControls.tsx`, `PlayerGameScreen.tsx`.

## Typography

All type is Cairo font via `font-[family-name:var(--font-cairo)]` (or `font-sans` which resolves to Cairo via Tailwind config).
RTL text is right-aligned via `dir="rtl"` on `<html>` — no `text-right` class ever used.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (regular) | 1.5 | Waiting copy, helper text, secondary labels (e.g., "في انتظار اللاعبين…", option letter labels) |
| Label | 16px (text-base) | 600 (semibold) | 1.5 | Option button text, free text submit button, voting answer card text, host control button labels |
| Heading | 24px (text-2xl) | 700 (bold) | 1.2 | Question counter ("سؤال N من T"), free text question text on player phone |
| Display | 48px (text-5xl) | 900 (black) | 1.1 | Question text on host screen |

Additional sizes in use (from codebase — maintain consistency):
- Audio badge text: 32px (text-4xl or text-3xl), weight 700 — "استمع وخمّن"
- Host option text in 2x2: 32px (text-[32px]), weight 700 — existing QuestionDisplay.tsx pattern
- Answer feed answer text on host: 20px (text-xl), weight 600 — large enough to read from across the room

Source: codebase read — `QuestionDisplay.tsx`, `AnswerOptions.tsx`, `WaitingScreen.tsx`, `HostGameScreen.tsx`. Layout.tsx confirms Cairo weights 400/600/700/900.

## Color

This project uses semantic Tailwind color names. No hex tokens. The 60/30/10 rule maps to the two contexts (host screen = dark; player phone = light).

### Host Screen (dark — landscape TV/PC)

| Role | Tailwind Value | Usage |
|------|----------------|-------|
| Dominant (60%) | `bg-gray-950` | Full host screen background |
| Secondary (30%) | `bg-gray-900` | Top bar, bottom controls bar, option button group containers |
| Accent (10%) | `bg-indigo-600` | Primary host CTA only: "اكشف الإجابة" button during question phase; audio pulse ring |

Accent reserved for (host screen): the single active CTA button only — "اكشف الإجابة" and "اغلق الإجابات" (lock free text). All other buttons use gray-700 or option colors.

### Player Screen (light — portrait mobile)

| Role | Tailwind Value | Usage |
|------|----------------|-------|
| Dominant (60%) | `bg-white` | Full player screen background |
| Secondary (30%) | `bg-gray-50` or `bg-gray-100` | Textarea container, voting answer card background, waiting screen container |
| Accent (10%) | `bg-indigo-600` / `text-indigo-600` | "أرسل" (send) submit button, focus rings (`focus-visible:ring-indigo-500`) |

Accent reserved for (player screen): "أرسل" button when enabled, timer bar, focus rings. Not used for waiting copy or decorative elements.

### Option Colors (game mechanic — not accent)

These are gameplay semantic colors, not accent usage:

| Option | Color |
|--------|-------|
| A | `bg-red-500 text-white` |
| B | `bg-blue-500 text-white` |
| C | `bg-yellow-400 text-gray-900` |
| D | `bg-green-500 text-white` |

### Semantic / State Colors

| State | Color | Usage |
|-------|-------|-------|
| Destructive | `bg-red-600 hover:bg-red-700` | "إنهاء اللعبة" (end game) host control only |
| Disabled | `bg-gray-700 opacity-40` | Inactive host controls; `"أرسل"` button when textarea is empty |
| Correct reveal | `brightness-110 ring-4 ring-white` | Correct option on reveal (host + player) |
| Wrong reveal | `opacity-20 saturate-0` | Wrong selected option on player screen |
| Dim others | `opacity-30` | Non-selected, non-correct options on reveal |

Source: codebase read — `HostGameScreen.tsx`, `HostInGameControls.tsx`, `AnswerOptions.tsx`, `PlayerGameScreen.tsx`, `WaitingScreen.tsx`.

## Interaction Contracts

### Media Guessing — Image Blur

| State | Visual |
|-------|--------|
| Question starts | `filter: blur(20px)`, `transition: filter {timerDuration * 1000}ms linear` — inline style on wrapper div |
| Timer running | CSS transition animates blur from 20px to 0px automatically |
| Host reveals early | `filter: blur(0px)`, `transition: filter 300ms ease-out` — fast snap |
| Revealed | No blur, full image visible |

Implementation: inline `style` attribute on wrapper `<div>`, not Tailwind class. Key the outer question container (not the blur div) on `questionIndex` to avoid mid-transition remount.

### Media Guessing — Audio

| State | Visual |
|-------|--------|
| Audio question starts | Hidden `<audio autoPlay loop src={mediaUrl} aria-label="مقطع صوتي للسؤال" />` |
| Autoplay succeeds | No visible indicator beyond badge |
| Autoplay fails (mobile) | Fallback tap button: `▶ استمع` — calls `audioRef.current.play()` — shown only if `onPlay` not fired within 300ms |
| Badge | Large emoji 🎵 with `animate-pulse` (existing Tailwind utility), text "استمع وخمّن" at 32px/700 |
| Options | Same 4-option layout as MC (same `AnswerOptions` reused) |

### Free Text — Player Input

| State | Visual |
|-------|--------|
| Timer running, not submitted | Textarea visible, placeholder "اكتب إجابتك هنا…", "أرسل" button disabled (opacity-40) |
| At least 1 character typed | "أرسل" button enabled (`bg-indigo-600`) |
| After submit | `FreeTextInput` hidden; `WaitingScreen` shown with submitted text greyed (`text-gray-400`) and spinner |
| Voting phase arrives | Voting UI replaces waiting screen |

### Free Text — Host Feed

| State | Visual |
|-------|--------|
| Timer running | `FreeTextFeed` visible; answers stream in via `freetext:answers` events |
| Each answer row | Emoji avatar (1.5rem) + answer text (text-xl/600) + entry animation via `motion/react` AnimatePresence slide-in |
| Feed order | Shuffled before display (server-side per D-08) |
| Max per answer | 80 chars (client enforced) |

### Free Text — Voting Phase

| State | Visual |
|-------|--------|
| Phase enters | `VotingDisplay` on host; `VotingUI` on player |
| Host `VotingDisplay` | All locked answers visible, 15s countdown timer, no interaction needed from host |
| Player `VotingUI` | Scrollable list of answer cards; own answer card: `opacity-40 cursor-not-allowed`; unvoted others: tappable, `hover:bg-gray-100 active:scale-95`; after vote: voted card gets `ring-4 ring-indigo-500`, others dim |
| Voting countdown | Shown on both host and player screens; mirrors `PlayerTimerBar` style (existing) — 15s fixed |
| Results reveal | `freetext:results` triggers: winning answer card highlighted (`bg-yellow-400 text-gray-900`), score overlay animates in (same `LeaderboardOverlay` pattern) |

## RTL Constraints (Mandatory — from Phase 3+4)

These rules are locked project-wide. Phase 5 components must follow them without exception.

| Rule | Detail |
|------|--------|
| ALWAYS use logical properties | `ms-`, `me-`, `ps-`, `pe-`, `text-start`, `text-end` |
| NEVER use physical properties | `ml-`, `mr-`, `pl-`, `pr-`, `text-left`, `text-right` |
| `dir="rtl"` on `<html>` | Already set in `layout.tsx` — all new components inherit |
| `dir="rtl"` on textarea | Explicit `dir="rtl"` + `lang="ar"` on `FreeTextInput` textarea |
| `<Image>` for Cloudinary | Must use `next/image` `<Image>` — never `<img>` tag |
| Font class | `font-[family-name:var(--font-cairo)]` or `font-sans` (both resolve to Cairo) |

Source: CONTEXT.md canonical refs + `apps/web/app/globals.css` RTL convention block.

## Animation Contract

All animation via `motion/react` (already installed). Respect `prefers-reduced-motion` via `useReducedMotion()` hook — existing pattern in `QuestionDisplay.tsx`.

| Element | Animation | Reduced Motion |
|---------|-----------|----------------|
| Free text answer appearing in feed | `AnimatePresence` + slide-in from start (x: -20 → 0, opacity: 0 → 1, 200ms ease-out) | Fade only (opacity: 0 → 1) |
| Voting answer card tap | `active:scale-95` Tailwind utility | No scale |
| Audio badge | `animate-pulse` Tailwind utility | Remove animate-pulse class |
| Image blur transition | CSS `transition: filter` — not controlled by motion/react | Not applicable (CSS transition, not JS animation) |
| Winning answer highlight on `freetext:results` | AnimatePresence scale + yellow background swap (300ms) | Opacity swap only |
