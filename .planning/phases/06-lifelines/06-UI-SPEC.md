## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Letter labels inside answer buttons, icon-to-label gap within lifeline button |
| sm | 8px | Gap between the three lifeline buttons, inline padding within compact UI |
| md | 16px | Horizontal padding on the lifeline row container, section spacing |
| lg | 24px | Padding on player-picker overlay content area |
| xl | 32px | Vertical gap between lifeline row and answer options block |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Lifeline buttons: minimum touch target 44px height (WCAG 2.1 SC 2.5.5, confirmed by ui-ux-pro-max skill rule `touch-target-size`)
- Player-picker overlay list items: minimum 48px height per row (ensures fat-finger safe on mobile portrait)
- Gap between adjacent lifeline buttons: 8px minimum (skill rule `touch-spacing`)

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|

## Color

Dark/light split: PlayerGameScreen uses `bg-white` (mobile portrait). Host screen uses `bg-gray-950`/`bg-gray-900`. Lifeline UI lives on the player controller — white surface.

| Role | Tailwind Token | Usage |
|------|---------------|-------|
| Dominant (60%) | white / bg-white | PlayerGameScreen background, lifeline row background |
| Secondary (30%) | gray-100 / bg-gray-100 | Player-picker overlay backdrop, inactive button surface |
| Accent (10%) | indigo-600 / bg-indigo-600 | Active lifeline buttons, primary CTA, timer bar (existing indigo-500) |
| Destructive | red-600 / bg-red-600 | Freeze Opponent lifeline when targeting; matches existing HostInGameControls "End Game" pattern |

Accent reserved for:
1. Active (unspent) lifeline buttons — all three when available
2. The timer progress bar (existing indigo-500 — do not change)

Selection in the Freeze Opponent overlay fires directly on player row tap — there is no separate Confirm button.

Spent state color: `bg-gray-200 text-gray-400` with `opacity-50` — matches the "other options greyed out" pattern in AnswerOptions.tsx (existing codebase pattern).

Answer option colors (unchanged from Phase 4):
- A: red-500
- B: blue-500
- C: yellow-400 (text-gray-900 for 8.4:1 contrast)
- D: green-500

Remove Two eliminated options: same `bg-gray-200 text-gray-400` spent pattern, plus `line-through` on text to signal elimination.

## Component Inventory

### 1. LifelineBar

Location: `apps/web/app/join/[roomCode]/game/LifelineBar.tsx`

A horizontal row of three lifeline buttons, placed between the PlayerTimerBar and the question header. The row must not displace the answer options — it should be compact (h-11 / 44px buttons).

Layout:
```
[ نقاط مضاعفة ] [ أزل خيارين ] [ جمّد منافس ]
```

Placement in PlayerJoin.tsx: rendered above the question header `<div className="px-4 pt-4 pb-2">` for MC/MEDIA_GUESSING questions. Not rendered during FREE_TEXT questions (lifelines only apply to MC and MEDIA_GUESSING).

Props:
```ts
interface LifelineBarProps {
  doublePointsUsed: boolean
  removeTwoUsed: boolean
  freezeOpponentUsed: boolean
  onDoublePoints: () => void
  onRemoveTwo: () => void
  onFreezeOpponent: () => void
  disabled: boolean  // true after player has answered (playerPhase !== 'answering')
}
```

Active button classes:
```
rounded-xl px-3 py-2 min-h-[44px] min-w-[44px] text-sm font-bold
bg-indigo-600 text-white active:scale-95 active:brightness-90
transition-all duration-150 cursor-pointer
focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2
```

Spent button classes (after use):
```
rounded-xl px-3 py-2 min-h-[44px] min-w-[44px] text-sm font-bold
bg-gray-200 text-gray-400 opacity-50 pointer-events-none cursor-not-allowed
```

Container:
```
flex flex-row gap-2 px-4 py-2 w-full
```

Each button carries `aria-label` and `aria-disabled` attributes. Spent buttons render with `aria-disabled="true"` and `disabled` HTML attribute.

### 2. FreezeOpponentOverlay

Location: `apps/web/app/join/[roomCode]/game/FreezeOpponentOverlay.tsx`

A full-screen overlay triggered when player taps "جمّد منافس". Shows a list of other players; player taps one to freeze them.

Structure:
```
Fixed full-screen dim backdrop (bg-gray-950/80 backdrop-blur-sm)
  └── Centered modal panel (bg-white rounded-2xl p-6 mx-4 max-w-sm w-full)
       ├── Heading: "اختر منافسًا للتجميد" (text-xl font-bold text-gray-900 text-center mb-4)
       ├── Player list (flex flex-col gap-2)
       │    └── PlayerRow × N (min-h-[48px] rounded-xl flex items-center gap-3 px-4
       │         bg-gray-100 active:bg-indigo-50 cursor-pointer transition-colors)
       │         ├── Emoji (text-2xl)
       │         └── Name (text-base font-bold text-gray-900)
       └── Cancel button (text-sm text-gray-400 underline mt-4 text-center w-full)
            copy: "إلغاء التجميد"
```

Props:
```ts
interface FreezeOpponentOverlayProps {
  players: Array<{ id: string; name: string; emoji: string }>
  onSelect: (playerId: string) => void
  onCancel: () => void
}
```

The overlay is portal-rendered at document body level (or as a sibling to PlayerGameScreen with `fixed inset-0 z-50`).

Accessibility:
- `role="dialog"` on the modal panel
- `aria-modal="true"`
- `aria-labelledby` pointing to the heading element
- Focus trapped inside overlay while open (focus first player row on open, Escape key fires onCancel)

## Interaction States

### Lifeline Button States

| State | Visual |
|-------|--------|
| Active (available) | bg-indigo-600 text-white, full opacity |
| Pressed | active:scale-95 active:brightness-90 (150ms transition) |
| Disabled (answered, not used) | bg-indigo-600 text-white opacity-40 pointer-events-none |
| Spent (used this game) | bg-gray-200 text-gray-400 opacity-50 pointer-events-none |
| Double Points — activated (before reveal) | ring-4 ring-indigo-300 ring-offset-2 bg-indigo-700 (indicates "active this question") |

"Disabled" (after answering) is distinct from "Spent" (used this game). Disabled is temporary per-question; Spent is permanent per-game.

### Double Points Active Indicator

When Double Points is activated but the question is not yet revealed, add a visible badge below the lifeline row:
```
<p className="text-xs text-indigo-600 font-bold text-center animate-pulse">
  النقاط مضاعفة لهذا السؤال ×2
</p>
```

Remove the badge after `question:revealed` fires.

### Remove Two Flow

1. Player taps "أزل خيارين"
2. Emit `lifeline:remove_two` socket event
3. Server responds with `lifeline:remove_two_result` → `{ eliminatedIndices: [n, m] }`
4. AnswerOptions re-renders with those indices greyed out + line-through
5. Lifeline button immediately enters Spent state (no round-trip needed — optimistic)

### Freeze Opponent Flow

1. Player taps "جمّد منافس"
2. FreezeOpponentOverlay opens with list of other players (from existing `players` state)
3. Player taps a target → emit `lifeline:freeze_opponent` with `{ targetPlayerId }`
4. Overlay closes immediately (optimistic)
5. Server confirms; if error, show error toast (see Copywriting section)
6. Lifeline button enters Spent state

### Overlay Dismiss

- Tap backdrop → dismiss (fires onCancel)
- Tap Cancel button → dismiss
- Press Escape → dismiss (keyboard/desktop accessibility)

## Copywriting Contract

All copy is Arabic. No English labels anywhere in the player controller.

| Element | Copy |
|---------|------|
| Double Points button label | نقاط مضاعفة |
| Remove Two button label | أزل خيارين |
| Freeze Opponent button label | جمّد منافس |
| Double Points active badge | النقاط مضاعفة لهذا السؤال ×2 |
| Freeze Opponent overlay heading | اختر منافسًا للتجميد |
| Freeze Opponent cancel | إلغاء التجميد |
| Freeze Opponent screen-reader label | اختر لاعبًا لتجميده لهذا السؤال |
| Spent lifeline aria-label suffix | (مستخدم) — appended to button aria-label, e.g. "نقاط مضاعفة (مستخدم)" |
| Error — already used (defensive) | تم استخدام هذا المساعد مسبقًا |
| Error — freeze failed (target already answered) | لم يتمكن التجميد من الوصول في الوقت المناسب |
| Error — freeze failed (no valid target) | لا يوجد منافسون متاحون للتجميد |
| Lifeline row section aria-label | مساعدات اللعبة |
| Empty lifelines (all spent, aria) | تم استخدام جميع المساعدات |
| Player not yet answered (lobby display) | في انتظار اللاعبين… (existing — no change) |

No destructive actions in this phase. Freeze Opponent targets an opponent but does not destroy data; no confirmation dialog beyond the player-picker overlay itself, which is already a confirmation step.

Error display: errors appear as a dismissing toast/banner below the lifeline row, 3-second auto-dismiss:
```
<div role="alert" className="mx-4 rounded-xl bg-red-50 border border-red-200 px-4 py-2 text-sm text-red-700 text-center">
  {errorMessage}
</div>
```

This matches the existing error treatment in PlayerJoin.tsx form (`bg-red-50 border border-red-200 text-red-700`).

## Animation Contract

| Interaction | Animation | Duration | Easing |
|-------------|-----------|----------|--------|
| Lifeline button press | scale-95 brightness-90 | 150ms | linear (active: pseudo-class) |
| Lifeline → Spent transition | opacity fade 0→50% | 300ms | ease-out |
| Double Points badge appear | fade-in (opacity-0 → opacity-100) | 200ms | ease-out |
| Overlay appear | opacity-0 → opacity-100, translateY(8px → 0) | 200ms | ease-out |
| Overlay dismiss | opacity-100 → opacity-0 | 150ms | ease-in |
| Error toast appear/dismiss | opacity-0 ↔ opacity-100 | 200ms | ease-out |

All animations must respect `prefers-reduced-motion`: wrap transition classes in a `motion-safe:` Tailwind variant or check `window.matchMedia('(prefers-reduced-motion: reduce)')` and apply instant state changes if true.

## Accessibility Contract

| Requirement | Implementation |
|-------------|----------------|
| Touch target minimum 44px | min-h-[44px] min-w-[44px] on all lifeline buttons |
| Touch target spacing | gap-2 (8px) between lifeline buttons |
| Color not the only differentiator | Each lifeline has unique Arabic text label (not icon-only) |
| Spent state communicates non-visually | aria-disabled="true" + disabled attribute + aria-label suffix "(مستخدم)" |
| Focus rings | focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2 on all interactive elements |
| Screen reader region | aria-label="مساعدات اللعبة" on the LifelineBar container |
| Overlay dialog | role="dialog" aria-modal="true" aria-labelledby on FreezeOpponentOverlay |
| Focus trap | Focus constrained to overlay while open; returns to Freeze button on close |
| Escape key | Fires onCancel on FreezeOpponentOverlay |
| Error announcements | role="alert" on error toast (screen reader announces immediately) |
| RTL logical properties | Use ms/me/ps/pe, text-start/text-end — never left/right |
