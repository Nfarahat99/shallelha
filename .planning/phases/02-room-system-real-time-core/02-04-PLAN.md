**Step 1: Install Artillery as devDependency.**

```bash
npm install -D artillery artillery-engine-socketio-v3 --workspace=apps/server
```

Per RESEARCH.md: `artillery-engine-socketio-v3` is required because the default Artillery Socket.io engine uses v2 client — incompatible with our Socket.io v4 server. The `v3` engine supports Socket.io v4 protocol.

**Step 2: Create `apps/server/load-test/100-rooms.yml` — Artillery config.**

Per RESEARCH.md load test pattern, with adjustments for realistic simulation:

```yaml
# Artillery load test: 100 simultaneous rooms
# Usage: npx artillery run load-test/100-rooms.yml --target <SERVER_URL>
#
# Expected results (INFRA-02):
# - 0 connection errors
# - p99 latency for room:join response < 200ms
# - 300 concurrent WebSocket connections sustained for 30s

config:
  target: "{{ $processEnvironment.TARGET_URL }}"
  phases:
    - name: "Ramp up to 100 rooms"
      duration: 30
      arrivalRate: 10
  engines:
    socketio-v3:
      transports:
        - websocket
  socketio:
    transports:
      - websocket

scenarios:
  - name: "Player joins room and holds connection"
    engine: socketio-v3
    flow:
      - emit:
          channel: "room:join"
          data:
            roomCode: "TEST"
            name: "لاعب تجريبي"
            emoji: "🦁"
      - think: 10
```

This ramps up 10 connections/second for 30 seconds = ~300 total connections. Each connection emits a `room:join` event and holds the connection for 10 seconds, simulating real player behavior.

NOTE: In a real load test, each connection should create or join a unique room. For Phase 2 MVP, this test validates that the Socket.io server can handle 300 concurrent WebSocket connections without dropping. A more sophisticated test (with room creation per connection) can be added in Phase 8.

**Step 3: Create `apps/server/load-test/run-load-test.sh` — runner script.**

```bash
#!/bin/bash
# Run Artillery load test against target URL
# Usage: ./load-test/run-load-test.sh <TARGET_URL>
#
# Example (local):   ./load-test/run-load-test.sh http://localhost:4000
# Example (staging):  ./load-test/run-load-test.sh https://shallelha-server-production.up.railway.app
#
# INFRA-02 pass criteria:
# - 0 connection errors
# - 300 concurrent WebSocket connections
# - p99 response time < 200ms

set -e

TARGET_URL="${1:-http://localhost:4000}"

echo "Running load test against: $TARGET_URL"
echo "Expected: 300 connections, 0 errors, p99 < 200ms"
echo ""

TARGET_URL="$TARGET_URL" npx artillery run load-test/100-rooms.yml

echo ""
echo "Load test complete. Review results above."
echo "Pass criteria: 0 errors in 'errors' section, p99 < 200ms in 'socketio.response_time'"
```

Make the script executable: `chmod +x apps/server/load-test/run-load-test.sh`

**Step 4: Add a convenience npm script to apps/server/package.json:**

Add to scripts: `"test:load": "bash load-test/run-load-test.sh"`
