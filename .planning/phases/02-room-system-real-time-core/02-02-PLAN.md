**Step 3: Create `apps/server/src/services/room.service.ts` — GREEN phase.**

Implement the room service using ioredis. Follow RESEARCH.md Pattern 5 exactly.

Key implementation details:

1. **Room code generation** — Use charset `ABCDEFGHJKLMNPQRSTUVWXYZ23456789` (32 chars, no I/O/0/1). Attempt up to 10 times with Redis `SET NX EX` on `room:{code}:lock` key for atomic collision check.

2. **createRoom(hostId)** — Generate code, then MULTI: `HSET room:{code}` with fields `{hostId, status: "waiting", createdAt: new Date().toISOString(), players: "[]"}`, `EXPIRE room:{code} 86400`. Return code.

3. **addPlayer(roomCode, { socketId, name, emoji })** — Atomic join using retry loop (max 3 attempts):
   - `WATCH room:{code}` 
   - `HGET room:{code} players` — parse JSON
   - If null: return `{ ok: false, error: "Room not found" }`
   - If players.length >= 8: `UNWATCH`, return `{ ok: false, error: "Room is full" }`
   - Generate `reconnectToken` using `crypto.randomUUID()`
   - Push new player: `{ socketId, reconnectToken, name, emoji, connectedAt: new Date().toISOString(), disconnectedAt: null }`
   - MULTI: `HSET room:{code} players JSON.stringify(updatedPlayers)`
   - EXEC — if null (contention), retry
   - Return `{ ok: true, reconnectToken, players: updatedPlayers }`

   > ⚠️ **CRITICAL — IGNORE RESEARCH.md Pattern 5.** Pattern 5 shows `redis.watch(key, async callback)` — this API does NOT exist in ioredis and will throw a runtime error. Use ONLY the sequential approach below:

   NOTE on ioredis WATCH pattern: ioredis does not have a callback-based `watch()`. Use the pipeline approach:
   ```typescript
   await redis.watch(key)
   const raw = await redis.hget(key, 'players')
   // ... validate and build new state
   const result = await redis.multi().hset(key, 'players', JSON.stringify(updated)).exec()
   if (result === null) { /* retry */ }
   ```

4. **markDisconnected(roomCode, socketId)** — HGET players, find by socketId, set disconnectedAt, HSET back.

5. **removePlayer(roomCode, socketId)** — HGET players, filter out by socketId, HSET back. Called after 10s grace period expires.

6. **reconnectPlayer(roomCode, reconnectToken, newSocketId)** — HGET players, find by reconnectToken, check `disconnectedAt` is within 10s of now. If valid: update socketId, clear disconnectedAt, HSET back. Return `{ ok: true, players }`. If expired or not found: `{ ok: false }`.

7. **getRoom(roomCode)** — HGETALL, parse players JSON, return typed RoomData or null.

8. **updateStatus(roomCode, status)** — HSET room:{code} status.

Run tests again: `cd apps/server && npx vitest run src/services/room.service.test.ts` — all should PASS.

**Step 2: Create `apps/server/src/socket/middleware/auth.ts`.**

Per RESEARCH.md Pattern 6:
```typescript
import type { Server } from 'socket.io'

export function registerSocketMiddleware(io: Server) {
  io.use((socket, next) => {
    const userId = socket.handshake.auth?.userId
    if (userId) {
      socket.data.userId = userId
    }
    next()
  })
}
```

Players connect without userId (anonymous). Hosts connect with userId from NextAuth session. The middleware stores it in `socket.data.userId` for later host verification.

**Step 3: Create `apps/server/src/socket/handlers/room.ts`.**

Per RESEARCH.md Pattern 4. Register handlers on the socket:

- **`room:create`** — `({ }) => { ... }`. Requires `socket.data.userId`. Call `roomService.createRoom(socket.data.userId)`. Join the Socket.io room. Emit `room:created` with `{ roomCode }`.

- **`room:join`** — `({ roomCode, name, emoji }) => { ... }`. Normalize code to uppercase. Call `roomService.addPlayer(code, { socketId: socket.id, name, emoji })`. If not ok, emit `room:error`. Otherwise: `socket.join(code)`, emit `room:joined` to joining player with `{ reconnectToken, players }`, broadcast `lobby:update` to room with `{ players }`.

- **`room:leave`** — `({ roomCode }) => { ... }`. Call `roomService.removePlayer(roomCode, socket.id)`. Leave Socket.io room. Broadcast `lobby:update`.

- **`room:reconnect`** — `({ roomCode, reconnectToken }) => { ... }`. Call `roomService.reconnectPlayer(code, reconnectToken, socket.id)`. If ok: `socket.join(code)`, emit `room:reconnected`, broadcast `lobby:update`. If not ok: emit `room:error` with "Reconnect failed — please rejoin".

- **`room:rejoin`** — `({ roomCode }) => { ... }`. Called by the HOST when they navigate directly to `/host/[roomCode]` or refresh the page (NOT a new room creation). Look up the existing room in Redis via `roomService.getRoom(code)`. Verify `room.hostId === socket.data.userId`. If match: call `socket.join(code)`, emit `lobby:update` with current players to this socket. If no match or room not found: emit `room:error` "Room not found or not authorized". This prevents creating a NEW room on host page refresh (BLOCKER — `room:create` always generates a new code).

- **`disconnecting`** — Read rooms from `socket.rooms` (filter out socket.id). For each room: call `roomService.markDisconnected(roomCode, socket.id)`. Set a 10-second `setTimeout` that calls `roomService.removePlayer(roomCode, socket.id)` and broadcasts `lobby:update` to the room. Use `io.to(roomCode).emit(...)` inside the timeout (not `socket.to()` since the socket is already disconnected).

import { Router } from 'express'
import { roomService } from '../services/room.service'

export const roomRouter = Router()

roomRouter.get('/:code', async (req, res) => {
  const code = req.params.code.toUpperCase()

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Player socket -> Room service | Untrusted player input (name, emoji, roomCode) |
| Host socket -> Game controls | Must verify host identity before allowing game:start/pause/end |
| Redis -> Room state | Read-modify-write race conditions on player list |
| REST client -> /room/:code | Room code enumeration |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-07 | Elevation of Privilege | game:start/pause/end | mitigate | Server checks `room.hostId === socket.data.userId` before processing host-only events; anonymous sockets have no userId |
| T-02-08 | Tampering | room:join race condition | mitigate | Redis WATCH/MULTI/EXEC with retry loop (max 3 attempts) prevents TOCTOU race that could add 9th player |
| T-02-09 | Tampering | Player name input | mitigate | Validate name length (1-15 chars) and emoji against whitelist in room:join handler before Redis write |
| T-02-10 | Information Disclosure | GET /room/:code enumeration | accept | Room codes expire in 24h; no sensitive data returned (only status + player count, no names); rate limiting deferred to Phase 8 |
| T-02-11 | Denial of Service | Rapid room creation | accept | No rate limit on room:create in Phase 2; mitigated by 24h TTL auto-cleanup; rate limiting in Phase 8 |
| T-02-12 | Spoofing | reconnectToken replay | mitigate | Server checks disconnectedAt timestamp; token only valid within 10s window; token is per-session (UUID) |
| T-02-13 | Repudiation | Room events without logging | accept | Console.log on connect/disconnect; structured logging deferred to Phase 8 |
</threat_model>
