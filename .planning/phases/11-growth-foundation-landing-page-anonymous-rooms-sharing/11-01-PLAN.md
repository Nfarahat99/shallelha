  <action>
Add the following to BOTH schema files (apps/web/prisma/schema.prisma AND apps/server/prisma/schema.prisma):

1. Extend the User model with profile fields:
   - displayName   String?
   - avatarEmoji   String?   @db.VarChar(10)
   - totalGamesPlayed Int    @default(0)
   - winCount      Int       @default(0)
   - bestStreak    Int       @default(0)
   - favoriteCategory String?
   - gameResults   PlayerGameResult[]

2. Add GameSession model:
   - id          String   @id @default(cuid())
   - roomCode    String
   - hostId      String
   - playedAt    DateTime @default(now())
   - categoryId  String?
   - categoryName String?
   - playerCount Int
   - winnerId    String?
   - results     PlayerGameResult[]
   - createdAt   DateTime @default(now())

3. Add PlayerGameResult model:
   - id            String      @id @default(cuid())
   - gameSessionId String
   - gameSession   GameSession  @relation(fields: [gameSessionId], references: [id], onDelete: Cascade)
   - userId        String
   - user          User         @relation(fields: [userId], references: [id], onDelete: Cascade)
   - playerName    String
   - playerEmoji   String?
   - score         Int
   - rank          Int
   - isWinner      Boolean      @default(false)
   - createdAt     DateTime     @default(now())

   @@unique([gameSessionId, userId])

IMPORTANT: The server schema already has Pack and PackQuestion models — do NOT remove them. Only ADD the new models and User field extensions.

Do NOT add a foreign key from GameSession.hostId to User — the host may be identified by Google sub ID which may not exist as a User row (NextAuth has no Prisma adapter, so User records are not auto-created). Store hostId as plain String (same pattern as Pack.createdBy).

Do NOT add a foreign key from GameSession.categoryId to Category — the category may be from a pack or may not exist. Store as plain String with categoryName for display.
