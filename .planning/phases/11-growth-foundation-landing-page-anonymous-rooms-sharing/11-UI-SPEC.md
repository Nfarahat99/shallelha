# Phase 11 — UI Design Contract

> Visual and interaction contract for Phase 11: Growth Foundation — Landing Page, WhatsApp Share, QR Code, Post-Game Screen, User Profiles, Game History.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (custom Tailwind-only) |
| Icon library | inline SVG only — no external icon library |
| Font | Cairo via next/font/google — CSS variable `--font-cairo`; weights 400/600/700/900; subsets: arabic + latin |

**Source:** `apps/web/tailwind.config.ts`, `apps/web/app/layout.tsx`

RTL convention: All directional utilities use Tailwind logical properties — `ms-*`/`me-*`/`ps-*`/`pe-*` only. Never `ml-*`/`mr-*`/`pl-*`/`pr-*`. Root `<html lang="ar" dir="rtl">` is set globally in layout.tsx. Every new screen must carry `dir="rtl"` explicitly on its outermost container.

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, avatar-to-name gaps |
| sm | 8px | Compact element spacing, button internal gaps, pill tab padding |
| md | 16px | Default element padding, card body padding, form field spacing |
| lg | 24px | Section padding, card outer padding, group gaps |
| xl | 32px | Layout gaps between major blocks, page horizontal padding on desktop |
| 2xl | 48px | Major section breaks on landing page |
| 3xl | 64px | Page-level section spacing on landing page |

Exceptions:
- Touch targets: minimum 44×44px (`min-h-[44px] min-w-[44px]`) — all interactive elements on mobile screens (buttons, links, toggles)
- QR code: minimum 200×200px display area
- Room code display: `text-5xl font-bold tracking-widest font-mono` — fixed size, not part of type scale

**Source:** Established in `apps/web/app/auth/signin/page.tsx` (44px touch target), `apps/web/app/host/[roomCode]/HostDashboard.tsx` (lobby and ended patterns), CONTEXT.md

## Typography

|------|------|--------|-------------|-------|

| Label | 14px | 600 | 1.4 | Section headings, stat labels, badge text, form labels, tab labels |

## Color

| Role | Value | Tailwind Token | Usage |
|------|-------|----------------|-------|
| Dominant (60%) | #030712 | `gray-950` | Page backgrounds, game screen backgrounds via `bg-gradient-to-b from-gray-950 via-brand-950 to-gray-900` |
| Secondary (30%) | rgba(255,255,255,0.10) | `white/10` | Cards, player list items, lobby panels, form inputs, modal overlays — all as glass cards with `bg-white/10 backdrop-blur-xl border border-white/10` |
| Accent (10%) | #4f46e5 | `brand-600` | Reserved exclusively for: primary CTAs, active tab/toggle state, answer confirmation highlight, score text, focus ring on inputs, rank badge on player post-game screen |
| Destructive | #ef4444 | `red-400` | Error messages only — `text-red-400`; no destructive action buttons in this phase |

Accent reserved for: primary CTA buttons ("ابدأ لعبة", "انضم للعبة", "شارك نتيجتك", "العب مجدداً"), active variant pill in ResultCard, focused input border (`focus:border-brand-500`), score value text (`text-brand-400`), share button in post-game player screen.

Landing page exception: Landing page hero uses `bg-gradient-to-b from-brand-600 via-purple-600 to-violet-700` (dark gradient). Feature/explainer sections below the fold use white/gray-50 backgrounds with dark text — this is the only place in the app with a light surface.

**Source:** `apps/web/tailwind.config.ts`, `apps/web/app/page.tsx`, `apps/web/app/host/[roomCode]/HostDashboard.tsx`, `apps/web/app/host/[roomCode]/game/ResultCard.tsx`

## Screen-by-Screen Interaction Contract

### 1. Landing Page (`/`) — UPDATE existing page

**Current state:** Page exists with hero, 3-step explainer, features, stats, CTA. Needs CTA copy update + OG meta.

- Layout: `max-w-2xl mx-auto` for content columns; `px-4 sm:px-6` gutters
- Hero: Dark gradient `from-brand-600 via-purple-600 to-violet-700` — keep as-is
- Brand: "شعللها" at `text-6xl sm:text-8xl font-black text-white`
- Tagline: Keep existing "أوقد المنافسة — منصة الألعاب الجماعية العربية"
- Primary CTA (host): **Update from "أنشئ غرفة" → "ابدأ لعبة"** — `bg-white text-brand-600 hover:bg-white/90 rounded-2xl px-8 py-4 text-lg font-bold min-h-[56px]`
- Secondary CTA (player): **Update from "عندك كود؟ انضم" → "انضم للعبة"** — `bg-white/15 border border-white/30 text-white hover:bg-white/25 rounded-2xl px-8 py-4 text-lg font-semibold min-h-[56px]`
- 3-step explainer: Keep existing — visual icons + Arabic step text
- OG meta tags: Add via Next.js `generateMetadata` — title `شعللها 🎮 — لعبة الأسئلة الجماعية`, description `ابدأ لعبة أسئلة مع أصدقائك في ثوانٍ — بدون تسجيل`

### 2. Host Lobby (`/host/[roomCode]`) — ADD WhatsApp + QR

**Location:** `HostDashboard.tsx` lobby state (`status === 'lobby'`)

- WhatsApp share button: Full-width button below room code card
  - Style: `w-full flex items-center justify-center gap-2 py-3 rounded-2xl bg-green-600 hover:bg-green-500 active:bg-green-700 text-white font-bold text-base transition-all`
  - Icon: WhatsApp inline SVG (white, 20×20)
  - Label: "مشاركة عبر واتساب"
  - Behavior: `window.open('https://wa.me/?text=' + encodeURIComponent(message))` — no API key needed
  - Message template (from CONTEXT.md): `انضم لجلسة شعللها 🎮\nكود الغرفة: ${code}\n${FRONTEND_URL}/join/${code}`
- QR code: Displayed in the room code card or below it
  - Size: 200×200px minimum, `rounded-xl` corners
  - Container: `bg-white p-3 rounded-xl mx-auto` (white background needed for QR readability)
  - Generation: `qrcode.toDataURL(joinUrl)` → `<img src={dataUrl} />` inline
  - Caption: `text-xs text-white/50 text-center mt-2` — "امسح للانضمام"
- Placement order in lobby: Header → Room code card (now includes QR) → WhatsApp button → Player list → Host controls

### 3. Deep Link Join (`/join/[roomCode]`) — UPDATE pre-fill

**Location:** `apps/web/app/join/[roomCode]/page.tsx` + `PlayerJoin.tsx`

- Room code input: Auto-populated from URL param when arriving via `/join/{code}`
- Player sees only name/emoji selection fields when code is pre-filled
- Input field: Same existing style — `rounded-xl border border-white/20 bg-white/10 text-white focus:border-brand-500`
- No visual difference from manual join; pre-fill is behavioral only

### 4. Post-Game Player Screen — NEW full screen

**Trigger:** Host emits `game:end` / socket `game:ended` event

**Layout:** `min-h-dvh flex flex-col bg-gradient-to-b from-gray-950 via-brand-950 to-gray-900 overflow-y-auto dir="rtl"` with `max-w-sm mx-auto w-full flex flex-col gap-6 px-6 py-8 pb-16`

**Sections:**
1. Header: "انتهت اللعبة!" at Heading size (24px/700), brand logo "شعللها" at Display size above
2. Rank badge (player's own rank): `rounded-2xl bg-white/10 border border-white/10 p-6 text-center`
   - Rank medal emoji: 🥇/🥈/🥉 or position number for rank > 3
   - Copy: "أنت في المرتبة الأولى" / "أنت في المرتبة الثانية 🥈" / "أنت في المرتبة الثالثة 🥉" / "المرتبة {N}"
   - Score: `text-brand-400 text-2xl font-bold` — "{score} نقطة"
3. Full leaderboard: All players, ranked
   - Each row: `flex items-center gap-3 px-4 py-3 rounded-2xl bg-white/8 border border-white/10`
   - Player's own row: highlighted with `bg-brand-600/20 border-brand-500/30`
   - Rank medal | Emoji avatar | Name + score column
4. Share button: "شارك نتيجتك" — `w-full py-3 rounded-2xl bg-brand-600 hover:bg-brand-500 text-white font-bold text-base min-h-[44px]`
   - Triggers Web Share API: `navigator.share({ title: 'شعللها', text: 'حصلت على المرتبة {N} في شعللها!' })`
   - Fallback: copy-to-clipboard if Web Share API unavailable
5. Auth prompt (if player was anonymous): `rounded-2xl bg-white/5 border border-white/10 p-4 text-center`
   - Copy: "سجّل دخولك لحفظ نتائجك"
   - Button: "تسجيل الدخول بـ Google" — `bg-white text-gray-900 rounded-2xl py-3 font-semibold`

### 5. Play Again Button — ADD to host ended state

**Location:** `HostDashboard.tsx` ended state

- Add "العب مجدداً" button above existing "العودة إلى الرئيسية" link
- Style: `w-full flex items-center justify-center py-3 rounded-2xl bg-brand-600 hover:bg-brand-500 text-white font-bold text-base min-h-[44px]`
- Behavior: Emits `game:reset` socket event → server resets Redis room → all clients return to lobby via `room:reset` socket event
- Placement: primary action above secondary "العودة" link

### 6. Shareable Result Cards (`/api/og/result`) — NEW edge route

- 1:1 (1200×1200) WhatsApp variant — existing ResultCard component already supports this via `variant='whatsapp'`
- 9:16 (1080×1920) Snapchat variant — existing ResultCard component already supports via `variant='snapchat'`
- NEW route: `apps/web/app/api/og/result/route.ts` using `@vercel/og` ImageResponse
- Content: top-3 leaderboard, player emojis, game stats, Sha'lelha logo, Arabic RTL text via Cairo font
- This is a parallel NEW route to the existing Express server route at `/cards/result` — the new one serves the OG preview image for `/join/{code}` link unfurls

### 7. OG Meta Tags for `/join/[roomCode]`

- `apps/web/app/join/[roomCode]/page.tsx`: Add `generateMetadata` export
- OG title: "انضم لجلسة شعللها 🎮" (from CONTEXT.md)
- OG description: "لديك دعوة للعب — اضغط للانضمام" (from CONTEXT.md)
- OG image: `/api/og/result?roomCode={code}` or static branded invite image
- `twitter:card`: `summary_large_image`

### 8. User Profile Page (`/profile`) — NEW screen

**Auth:** Google OAuth required — unauthenticated users redirect to `/auth/signin`

**Layout:** `min-h-dvh bg-gradient-to-b from-gray-950 via-brand-950 to-gray-900` with `max-w-lg mx-auto w-full flex flex-col gap-6 px-6 py-8`

**Sections:**
1. Avatar + name header: `rounded-2xl bg-white/10 border border-white/10 p-6`
   - Avatar: Large emoji at `text-6xl` centered
   - Display name: Heading size (24px/700) `text-white`
   - Edit button: `text-brand-400 text-sm font-semibold` inline link
2. Stats grid: 2×2 grid, each cell `rounded-2xl bg-white/8 border border-white/10 p-4 text-center`
   - Games played: `text-brand-400 text-2xl font-bold` + Label size "مباريات"
   - Wins: same pattern — "فوز"
   - Best streak: same pattern — "أطول سلسلة"
   - Favorite category: category name at Label size — "الفئة المفضلة"
3. Game history list: `space-y-3`
   - Each entry: `flex items-center gap-3 px-4 py-3 rounded-2xl bg-white/8 border border-white/10`
   - Date (Label size, `text-white/50`) | Category name | Score + rank (`text-brand-400`)
   - Empty state: "لم تلعب أي مباراة بعد" + "ابدأ لعبة الآن" link

### 9. Avatar Emoji Selector (profile edit)

- Reuse existing emoji picker pattern from `apps/web/components/ui/EmojiPicker.tsx`
- Grid layout of emoji options, tappable circles
- Selected emoji: `ring-2 ring-brand-500 bg-brand-600/20` highlight
- Modal or inline expansion triggered by "تعديل" button on profile
