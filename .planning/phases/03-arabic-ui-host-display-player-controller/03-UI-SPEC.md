## Typography

Two contexts — HOST screen (TV distance) and PLAYER screen (hand distance). Both use Cairo.

### Host Screen (TV/PC — read from ~2 metres)

| Role | Size | Tailwind | Weight | Cairo Weight | Line Height | Usage |
|------|------|----------|--------|--------------|-------------|-------|
| Display | 48px | text-5xl | Black | 900 | 1.2 | Question text on host screen |
| Heading | 32px | text-4xl (≈36px, use text-3xl=30px or custom text-[32px]) | Bold | 700 | 1.2 | Answer option text in 2×2 grid |
| Label | 24px | text-2xl | SemiBold | 600 | 1.3 | Leaderboard player names, timer digit |
| Caption | 16px | text-base | Regular | 400 | 1.5 | Room code corner display, metadata |

> Host text must be readable from across a room — minimum question size is 48px. Answer options at 32px minimum.

### Player Screen (mobile — hand distance)

| Role | Size | Tailwind | Weight | Cairo Weight | Line Height | Usage |
|------|------|----------|--------|--------------|-------------|-------|
| Heading | 20px | text-xl | Bold | 700 | 1.3 | Question text shown above answer buttons |
| Body | 16px | text-base | SemiBold | 600 | 1.4 | Answer button labels |
| Label | 14px | text-sm | Regular | 400 | 1.5 | Waiting spinner text, score display, status copy |
| Caption | 12px | text-xs | Regular | 400 | 1.5 | Player count, streak indicator |

**Font weight constraint:** Use exactly 2 weights per screen context.
- Host: 700 (Bold) + 900 (Black)
- Player: 400 (Regular) + 600/700 (SemiBold/Bold)

Source: CONTEXT.md specifics ("large text, high contrast, no small UI elements"; "Arabic question text must be right-aligned, font-weight 700+ for visibility on TV"), layout.tsx Cairo weight config [400, 600, 700, 900]

## Color

### Palette

The existing codebase establishes indigo as the established brand accent (indigo-600 for CTAs, indigo-50 for tinted cards). This phase extends that palette with answer-option color coding and game state semantics.

| Role | Tailwind Token | Hex (approx) | Usage |
|------|---------------|--------------|-------|
| Dominant (60%) | bg-gray-950 / bg-slate-900 | #030712 / #0f172a | Host screen full background (dark for TV contrast) |
| Dominant (60%) player | bg-white | #ffffff | Player mobile screen background |
| Secondary (30%) | bg-gray-800 / bg-slate-800 | #1e293b | Host screen card zones (question area, controls panel) |
| Secondary (30%) player | bg-gray-50 | #f9fafb | Player answer button base, waiting screen surface |
| Accent (10%) | indigo-600 (#4f46e5) | #4f46e5 | Primary CTA buttons, selected state ring, progress fill |
| Destructive | red-600 (#dc2626) | #dc2626 | End game button only (matches existing HostControls pattern) |
| Success semantic | green-500 (#22c55e) | #22c55e | Correct answer reveal highlight |
| Wrong semantic | gray-600 (#4b5563) at 30% opacity | — | Wrong answer dimmed state on reveal (opacity-30) |
| Answered indicator | indigo-400 (#818cf8) | #818cf8 | Player emoji avatar glow on host screen when answered |

### Answer Option Color Coding (Host 2×2 Grid)

Four distinct colors for the 4 answer quadrants — required for Kahoot-style orientation. Must pass 4.5:1 contrast with white text on TV at viewing distance:

| Option | Color | Tailwind | Hex | Contrast ratio (est.) |
|--------|-------|----------|-----|----------------------|
| A (top-start) | Red | bg-red-500 | #ef4444 | 4.5:1 with white |
| B (top-end) | Blue | bg-blue-500 | #3b82f6 | 4.8:1 with white |
| C (bottom-start) | Yellow | bg-yellow-400 | #facc15 | use text-gray-900 | 8.4:1 with near-black |
| D (bottom-end) | Green | bg-green-500 | #22c55e | 4.6:1 with white |

> Yellow option (C) MUST use `text-gray-900` not white — white on yellow-400 fails contrast.
> These 4 colors are for answer identity only. They are NOT used elsewhere in the UI.

### 60/30/10 Split Summary

**Host screen:**
- 60%: Dark background (bg-gray-950) — the "stage" the game plays on
- 30%: Mid-dark surfaces (bg-gray-800) — question zone, controls overlay
- 10%: Accent indigo + answer option colors — interactive elements only

**Player screen:**
- 60%: White background — clean, minimal, focus on buttons
- 30%: Light gray surfaces (bg-gray-50/100) — answer buttons unselected state
- 10%: Indigo accent — selected answer highlight, timer bar fill

**Accent reserved for:**
1. Primary CTA buttons (ابدأ اللعبة, اكشف الإجابة, التالي)
2. Selected answer button ring/border on player screen
3. Timer bar fill color
4. Player answered indicator glow on host screen
5. Indigo badge on host controls (room code card — existing pattern)

Source: CONTEXT.md D-01 to D-09, existing HostDashboard.tsx (bg-indigo-50, bg-indigo-600 pattern), HostControls.tsx (red-600 for end game), RESEARCH.md RTL-05 accessibility notes

These are non-negotiable for every component in this phase:

## Layout Contracts

### Host Screen (TV/PC — 16:9 landscape)

```
┌─────────────────────────────────────────────────────────┐
│  [Room Code — corner, caption size]    [Question N/N]   │  ← top bar, h-12, bg-gray-900
├─────────────────────────────────────────────────────────┤
│  [TIMER — full-width bar OR circle OR number]           │  ← timer zone, h-12 or absolute
├─────────────────────────────────────────────────────────┤
│                                                         │
│         QUESTION TEXT (text-5xl, weight 900)            │  ← question zone, min 40vh
│                (Arabic, text-start, RTL)                │
│                                                         │
├─────────────────────────────────────────────────────────┤
│  Layout variant A (2×2 grid — DEFAULT):                 │
│  ┌──────────────┐  ┌──────────────┐                     │
│  │  A — red-500 │  │  B — blue-500│                     │
│  └──────────────┘  └──────────────┘                     │
│  ┌──────────────┐  ┌──────────────┐                     │
│  │C — yellow-400│  │  D — green-500│                    │
│  └──────────────┘  └──────────────┘                     │
├─────────────────────────────────────────────────────────┤
│  [Player emoji indicators — flex row, avatars glow]     │  ← indicator bar, h-20
├─────────────────────────────────────────────────────────┤
│  [Reveal Answer]  [Next]  [Leaderboard]  [End Game]     │  ← host controls, h-16
└─────────────────────────────────────────────────────────┘
```

- Full screen: `w-screen h-screen overflow-hidden`
- No scroll on host screen — all content must fit in viewport
- Background: `bg-gray-950`
- Room code: positioned `absolute top-4 end-4` (RTL-safe — end = left in RTL)
- Question zone: `flex-1 flex items-center justify-center px-16 py-8`

### Host Layout Variants

**2×2 Grid (default):**
- `grid grid-cols-2 gap-4` in options zone
- Each cell: `rounded-2xl flex items-center justify-center min-h-[20vh] text-white font-bold text-3xl text-center p-6`
- Yellow cell: `text-gray-900` (not white — contrast)

**4-Column Bar:**
- `flex flex-row gap-3` at bottom of screen
- Each cell: `flex-1 rounded-xl min-h-[80px] flex items-center justify-center text-white font-bold text-xl`

**Vertical Stack:**
- Two-column layout: question takes `w-2/3`, options take `w-1/3` as `flex flex-col gap-3`
- Options: `rounded-xl py-4 px-6 flex items-center gap-3 text-white font-bold text-xl`

### Pre-Game Settings Screen (Host)

Displayed between lobby "Start Game" tap and game load. Host configures:

1. **Layout picker** — 3 radio-style cards showing visual preview (2×2 / 4-col / vertical)
2. **Timer style picker** — 3 options (bar / circle / number)
3. **Reveal mode toggle** — toggle switch: "اكتشاف تلقائي" / "اكتشاف يدوي"
4. **Start Game button** — `bg-indigo-600` primary CTA

Layout: centered modal or full-screen overlay on dark bg, max-w-2xl, p-8

## Interaction States

### Answer Buttons (Player Screen)

| State | Visual |
|-------|--------|
| Idle | bg-[option-color] opacity-100, white text, rounded-2xl |
| Pressed | scale-95, brightness-90 (CSS active pseudo) |
| Selected (post-tap) | ring-4 ring-white ring-offset-2, opacity-100, other options opacity-30 |
| Waiting (post-tap) | selected stays highlighted, others bg-gray-200 text-gray-400 pointer-events-none |
| Correct reveal | scale-105, brightness-110, check icon appears |
| Wrong reveal | opacity-20, saturate-0 |

### Answer Options (Host Screen — reveal)

| State | Visual |
|-------|--------|
| Active question | All 4 options full opacity, no indication of correct |
| Reveal — correct | `animate` scale 1.05 + ring-4 ring-white + check mark overlay |
| Reveal — wrong | `animate` opacity-30, scale 1.0 |
| Transition duration | 300ms (Motion `duration: 0.3`) |

### Player Avatar Indicators (Host Screen)

| State | Visual |
|-------|--------|
| Not answered | emoji shown, bg-gray-800 rounded-full, opacity-60 |

| Transition | 200ms spring on ring appearance |

| Button | Label | Style | Active when |
|--------|-------|-------|-------------|

| Show Leaderboard | عرض النتائج | bg-gray-700 | Phase = 'reveal' |

## Animation Contract

All animations use Motion (LazyMotion + domAnimation). `prefers-reduced-motion` is respected:
- If `prefers-reduced-motion: reduce` is active, remove all scale/translate transitions, keep opacity-only fade (150ms).

| Element | Animation | Motion Config |
|---------|-----------|---------------|
| Answer reveal (correct) | scale 1→1.05 + opacity unchanged | `duration: 0.3, ease: 'easeOut'` |
| Answer reveal (wrong) | opacity 1→0.3 | `duration: 0.3` |
| Player avatar answered | ring appears + scale 1→1.1→1 | `type: 'spring', stiffness: 400, damping: 20` |
| Leaderboard slide-in | x: '100%'→0, exit x: 0→'100%' | `type: 'spring', stiffness: 300, damping: 30` |
| Podium entrance (3→2→1) | y: 40→0, opacity: 0→1, stagger 0.4s | `staggerChildren: 0.4, staggerDirection: -1` |
| Timer bar | scaleX: 1→0 over timerDuration | CSS transition (not Motion) — `transition: transform Xs linear` |
| Question text entrance | opacity: 0→1, y: 16→0 | `duration: 0.25, ease: 'easeOut'` |

**Timer bar RTL:** Use `transform-origin: inline-end` so scaleX(0) depletes from start side in RTL (right→left).

Source: RESEARCH.md Pattern 4, CONTEXT.md D-05, key_context Motion recommendation

## Screen States Reference

### Host Screen State Machine

```
lobby → pre-game → question → reveal → leaderboard → question (loop)
                                                   ↘ podium (game end)
```

| Phase | What Host Sees | Controls Visible |
|-------|---------------|-----------------|
| lobby | Player list, room code, start button | ابدأ اللعبة |
| pre-game | Layout/timer/reveal pickers | حفظ وابدأ |
| question | Question + options + timer + player indicators | اكشف الإجابة |
| reveal | Correct answer highlighted, wrong dimmed | التالي / عرض النتائج / إنهاء |
| leaderboard | Sorted player list with scores | التالي / إنهاء |
| podium | Top 3 animated entrance | (game over — no further control) |

### Player Screen State Machine

```
form → lobby → question → waiting → (loop to question) → ended
```

| Phase | What Player Sees | Interaction |
|-------|-----------------|-------------|
| form | Name input + emoji picker | Join form submit |
| lobby | Other players + waiting indicator | None — wait |
| question | Question text + 4 answer buttons + timer bar | Tap one answer |
| waiting | Chosen answer highlighted, spinner | None — wait |
| ended | "انتهت اللعبة" + play again link | Tap replay |

## Accessibility Requirements

- All interactive elements: `focus-visible:ring-2 focus-visible:ring-indigo-500 focus-visible:ring-offset-2`
- Answer buttons: `aria-pressed` when selected; `aria-disabled` when post-answer
- Player indicator avatars: `aria-label="{playerName} — أجاب"` when answered
- Timer bar: `role="progressbar" aria-valuenow={remaining} aria-valuemax={duration} aria-label="الوقت المتبقي"`
- Host control buttons: descriptive `aria-label` matching button text
- Color not used as sole differentiator: answer options A/B/C/D labeled by letter in addition to color
- `prefers-reduced-motion`: disable all Motion animations; keep instant opacity changes

Source: SKILL.md ui-ux-pro-max, Priority 1 Accessibility, Priority 2 Touch & Interaction
