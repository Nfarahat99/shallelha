**Helper: `sendQuestion`** — sends question:start event:
```typescript
function sendQuestion(io: Server, roomCode: string, gameState: GameState, question: QuestionData) {
  const payload = {
    question: { text: question.text, options: question.options, timerDuration: question.timerDuration },
    questionIndex: gameState.currentQuestionIndex,
    total: gameState.questionIds.length,
    hostSettings: gameState.hostSettings,
  }
  io.to(roomCode).emit('question:start', payload)

  // Reset all players' answeredCurrentQ to false
  for (const id of Object.keys(gameState.playerStates)) {
    gameState.playerStates[id].answeredCurrentQ = false
  }

  // Auto-reveal timer if mode is 'auto'
  if (gameState.hostSettings.revealMode === 'auto') {
    clearAutoRevealTimer(roomCode)
    const timer = setTimeout(async () => {
      await handleReveal(io, roomCode)
    }, question.timerDuration * 1000)
    autoRevealTimers.set(roomCode, timer)
  }
}
```

**Event: `player:answer`** — Player submits an answer:
- Payload: `{ answerIndex: number }`
- Get roomCode from `socket.data.roomCode`
- Get gameState from Redis
- Validate: `gameState.phase === 'question'` — if not, silently return (no error to player per D-06)
- Validate: `answerIndex >= 0 && answerIndex <= 3` — if not, emit `room:error`
- Get player id from `socket.data.reconnectToken`
- Validate: `!gameState.playerStates[playerId]?.answeredCurrentQ` — if already answered, silently return
- Calculate elapsed: `Date.now() - gameState.questionStartedAt`
- Fetch correct answer: look up question from a module-level question cache (populated at game:start) or fetch from DB by id
- **Optimization:** Store question data (correctIndex, timerDuration) in a module-level `Map<string, QuestionData[]>` keyed by roomCode, populated at `game:start`. This avoids DB hits per answer.
- Calculate score: `calculateScore(answerIndex === correctIndex, elapsed, timerDurationMs, streak)`
- Update playerState: `score += points`, `answeredCurrentQ = true`, `lastAnswerTime = Date.now()`, and update streak: if correct `streak++`, else `streak = 0`
- Save GameState to Redis
- **Broadcast `question:progress` to room with the accumulated list of answered player IDs (required by D-03 for PlayerIndicators):**

**Helper: `handleReveal`** — shared between manual and auto reveal:
```typescript
async function handleReveal(io: Server, roomCode: string) {
  clearAutoRevealTimer(roomCode)
  const gameState = await getGameState(roomCode)
  if (!gameState || gameState.phase !== 'question') return
  
  gameState.phase = 'reveal'
  await saveGameState(roomCode, gameState)
  
  const questionData = questionCache.get(roomCode)?.[gameState.currentQuestionIndex]
  io.to(roomCode).emit('question:revealed', {
    correctAnswerIndex: questionData?.correctIndex ?? 0,
    playerResults: Object.entries(gameState.playerStates).map(([id, state]) => ({
      id,
      score: state.score,
      streak: state.streak,
    })),
  })
}
```

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| player socket -> game handler | Player can only submit answers — not reveal, advance, or end game |
| host socket -> game handler | Host can control game flow but cannot manipulate scores directly |
| client -> server timing | Player's answer timestamp is server-computed (Date.now()), not client-supplied |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-03 | Elevation of Privilege | game:* host events | mitigate | `requireHost()` check on game:configure, game:start, question:reveal, leaderboard:show, question:next, game:end |
| T-03-04 | Tampering | player:answer | mitigate | Server checks: `gameState.phase === 'question'`, `answerIndex in 0-3`, `!answeredCurrentQ`; elapsed computed server-side |
| T-03-05 | Tampering | player:answer timing | mitigate | `elapsedMs` = `Date.now() - gameState.questionStartedAt` — server timestamps only, client cannot influence score |
| T-03-06 | Repudiation | player:answer after reveal | mitigate | Server rejects answers when phase !== 'question'; silent rejection per D-06 |
| T-03-07 | Denial of Service | game:start | mitigate | Questions limited to 20 max; questionCache cleaned on game:end |
</threat_model>
